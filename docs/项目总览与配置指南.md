**项目概览**

- `market-backend`：BFF 网关，向前端提供 `HTTP/SSE` 接口，向下游通过 `gRPC` 访问 `market-service` 与推荐服务。
- `market-service`：核心业务服务，直连 `PostgreSQL` 与 `Redis`，同时调用推荐服务 `gRPC`。
- `block-listener`：区块链监听服务（如 Arbitrum），解析链上事件并回写至 `market-service`。
- `recommendation`：推荐/向量服务（Python），提供 embedding 与热门检索 `gRPC`。
- `market-proto`：统一收集并生成各服务的 `proto` 代码，作为跨服务依赖。

**目录结构**

- 根目录：`docker-compose.yml`（统一编排）、`configs/`（外置配置）
- 服务目录：`market-backend/`、`market-service/`、`block-listener/`、`recommendation/`、`market-proto/`
- SQL：`sql/`（各服务数据库脚本，链上事件使用独立库 `chain_events`）
- 重要文档：各服务 `README.md`、`SSE接口文档.md`（位于 `market-backend/`）

**端口与依赖**

- `market-backend`：`8000`，依赖 `market-service:9000` 与 `recommendation:50051`
- `market-service`：`9000`，依赖 `PostgreSQL:5432`、`Redis:6379`、`recommendation:50051`
- `block-listener`：`8080`，依赖 `market-service:9000`、区块链 RPC（Ethereum Sepolia）、`Redis`，数据库使用 `chain_events`
- `recommendation`：`50051`，依赖 `PostgreSQL`

**快速启动（Docker 根编排）**

- 前置条件：本机提供 `PostgreSQL` 与 `Redis`，容器通过 `host.docker.internal` 访问宿主机
  - PostgreSQL：示例库名 `usercenter`、`marketcenter`、`embedding`；用户与密码需与 `configs/*.yaml` 或 `.env` 对齐
  - Redis：默认无密码，`db0` 用于用户中心，`db1` 用于市场中心
- 在仓库根目录执行：
  - `docker compose up -d --build`
- 端口映射：
  - `127.0.0.1:8000 -> market-backend`
  - `127.0.0.1:9000 -> market-service`
  - `127.0.0.1:8080 -> block-listener`
  - `127.0.0.1:50051 -> recommendation`
- 配置挂载：
  - `./configs/market-backend:/app/configs`
  - `./configs/market-service:/app/configs`
  - `./configs/block-listener:/app/configs`

**单服务启动**

- Docker（进入对应服务目录）：`docker compose up -d --build`
- 本地运行（Go 服务）：
  - 安装依赖：`make init`
  - 生成代码：`make api`
  - 构建：`make build`
  - 启动：`make run`
- 本地运行（Python `recommendation`）：
  - 安装：`pip install -r requirements.txt`
  - 启动：`python main.py`

**配置文件说明（configs/\*.yaml）**

- 通用约定

  - 所有地址默认指向宿主机：`host.docker.internal`
  - 时间单位：如 `3s`、`3600s`
  - 为空的敏感信息（如 `api_key`、`verification_key`）需按需填写
  - `redis.db` 为逻辑库索引（0-15），用于隔离不同业务的 key 空间

- `configs/market-backend/config.yaml`

  - `server.http.addr`：HTTP 服务监听地址，默认 `:8000`
  - `data.redis.*`：Redis 连接（默认 `host.docker.internal:6379`，`db=0`）
  - `data.rpc.usercenter_rpc.addr`、`marketcenter_rpc.addr`：上游 `market-service` 的 `gRPC` 地址（默认 `host.docker.internal:9000`）
  - `data.privy.verification_key`：Privy JWT 验签公钥（PEM/ES256），用于校验前端令牌
  - `data.privy.app_id`：Privy 应用 ID（参与 JWT 的 `aud` 校验与 API 访问）
  - `data.privy.app_secret`：Privy 应用密钥（仅用于服务端访问 Privy API，不参与 JWT 验签）
  - `data.openai.base_url`：OpenAI/兼容服务商基础地址（建议统一在配置文件管理）
  - `data.openai.model/temperature/...`：推理参数
  - `custom.env`：环境标识（如 `dev`）
  - `custom.need_auth`：是否开启鉴权（`true/false`）
  - `custom.lark_*`：飞书告警配置（`webhook_key`、`enabled`）
  - `custom.asset_tokens.*`：资产代币元信息（如 `points`、`usdc` 的 `address/symbol/decimals`）

- `configs/market-service/config.yaml`

  - `server.grpc.addr`：`gRPC` 监听地址，默认 `:9000`
  - `data.postgresql.usercenter.dsn`：用户中心库连接串（默认 `usercenter`）
  - `data.postgresql.marketcenter.dsn`：市场中心库连接串（默认 `marketcenter`）
  - `data.redis.usercenter.*`：用户中心 Redis（`db=0`）
  - `data.redis.marketcenter.*`：市场中心 Redis（`db=1`）
  - `data.alchemy.api_key`：Alchemy API Key
  - `data.alchemy.network`：网络名称（如 `sepolia`）
  - `data.alchemy.chain_id`：链 ID（Sepolia：`11155111`）
  - `data.alchemy.policy_id`：Gas Manager 策略 ID（用于 `pm_*` 与 `alchemy_requestGasAndPaymasterAndData`）
  - `data.alchemy.entrypoint_address`：EIP-4337 EntryPoint 地址（`eth_sendUserOperation` 等调用）
  - `data.alchemy.fallback_address`：备用地址（当前代码未使用，可留空）
  - `data.alchemy.sign_tx_from_address`：服务端签名交易的 from 地址（当前代码未使用，可留空）
  - `data.alchemy.sign_tx_private_key`：服务端私钥（仅当需要服务端主动签 EOA 交易时必填）
  - `data.s3.*`：对象存储配置（`access_key`、`secret_key`、`region`、`bucket` 等）
  - `data.rpc.recommendation_rpc.addr`：推荐服务地址（默认 `host.docker.internal:50051`）
  - `custom.asset_tokens.*`：资产代币元信息
  - `custom.new_user_points`：新用户默认积分
  - `custom.chain`：链标识（如 `sepolia`）
  - `custom.lark_*`：飞书告警配置
  - `custom.min_volume`、`custom.max_deadline_hours`：交易最小量与最大截止小时数

- `configs/block-listener/config.yaml`
  - `data.db.dsn`：监听服务使用的数据库（已改为独立 `chain_events` 库）
  - `data.redis.*`：Redis 连接信息
  - `data.marketcenter_rpc.addr`、`usercenter_rpc.addr`：`market-service` 的 `gRPC` 地址
  - `data.blockchain.arb_rpc_urls`：Sepolia RPC 多地址（字段名保持不变）
  - `data.blockchain.start_block`：起始区块高度
  - `data.blockchain.factory_contract_address`：工厂合约地址
  - `data.blockchain.points_token_address`：积分代币地址
  - `data.blockchain.batch_scan_size/confirmed_block_num/cron_scan_db_rows`：批量扫描与确认策略
  - `custom.lark_*`、`custom.asset_tokens.*`：飞书告警与资产代币配置

**关键字段使用说明**

- `redis.db`：
  - 逻辑库索引（0-15），用于隔离不同业务的 key 空间
  - 代码读取：`src/market-service/internal/data/base/redis.go:32-39`
- `alchemy.sign_tx_private_key`：

- `privy.verification_key`：
  - 用于校验前端携带的 Privy JWT（ES256）
  - 验签实现：`src/market-backend/internal/pkg/middleware/privy_auth.go:84-113`

**服务关系与调用链**

- 前端 → `market-backend`（HTTP/SSE）
- `market-backend` → `market-service`（`gRPC`）
- `market-service` → `PostgreSQL/Redis`、→ `recommendation`（`gRPC`）
- 区块链事件 → `block-listener` → `market-service`（`gRPC`）

**常见问题与排错**

- 数据库不可达：确认宿主机 PostgreSQL 开放 `5432`，账号/库名与配置一致；Windows/Mac 使用 `host.docker.internal`。
- Redis 不通：确认 `6379` 端口与密码/DB 索引；必要时在 `configs` 中调整。
- OpenAI/Privy/Lark 为空：如未用到，可保持空；启用相关功能需填入真实密钥。
- 替换 OpenAI 兼容服务商：设置 `OPENAI_BASE_URL`，例如 `https://api.moonshot.cn/v1` 或 `https://api.deepseek.com/v1`
- 或在配置文件中设置 `data.openai.base_url` 统一管理
- 推荐服务未启动：`market-service` 调用推荐服务会失败，先确保 `50051` 可达或暂时关闭相关调用路径。
- 链上 RPC 限流/不可用：在 `block-listener` 填入多个 `arb_rpc_urls` 以容灾，适当调小批量参数。
